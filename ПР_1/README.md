# Практическая работа 1. Обработка и анализ данных с использованием Apache Spark (PySpark)

## Описание задачи и выбранного варианта

**Вариант:** 30. Документооборот

**Предметная область:** Анализ бизнес-процессов на основе логов обработки заявок/документов. Данная работа имитирует реальный процесс рассмотрения заявок (например, кредитных или иных документов) с последовательными этапами, временными метками, исполнителями и статусами выполнения.

**Цель работы:** Выявить «узкие места» процесса — этапы, на которых документы задерживаются дольше всего, а также обнаружить аномалии в ходе выполнения процесса с использованием методов статистического анализа (метод 3-сигм) и инструментов визуализации.

**Бизнес-задачи:**
1. Загрузить и обработать журнал обработки заявок (Access Logs) в HDFS
2. Подготовить данные для анализа: очистка, преобразование временных меток, вычисление длительности этапов
3. Выявить узкие места процесса — этапы с наибольшей средней длительностью
4. Обнаружить аномально долгие этапы 
5. Визуализировать результаты с помощью гистограмм, ящиков с усами (boxplot) и других графиков
6. Сформулировать рекомендации по оптимизации бизнес-процесса

## Инструкция по запуску

### 1. Инфраструктура и требования

**Системные требования:**
- OS: Ubuntu 20.04/24.04 LTS
- Hadoop: 3.3.5 (HDFS, YARN)
- Apache Spark: 3.4.3
- Java: OpenJDK 11
- Python: 3.12+

**Инструменты разработки:**
- **JupyterHub** — интерактивная разработка и выполнение ноутбуков
- **HDFS CLI** — управление файлами в распределенной файловой системе
- **Spark UI** (порт 4040) — мониторинг выполнения Spark-заданий
- **YARN UI** (порт 8088) — управление ресурсами кластера и мониторинг приложений
